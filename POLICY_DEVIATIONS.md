# POLICY_DEVIATIONS

## 1) UI/UX SSOT 대비 레거시 편차
- 정책: 공용 버튼 컴포넌트만 사용, 페이지 단위 인라인 스타일 금지.
- 현황: 기존 레거시 페이지 다수에서 `<button>` 직접 사용 및 인라인 스타일이 이미 광범위하게 사용 중입니다.
- 조치: 이번 작업에서는 기능 안정화(인증 루프/API 정합성/실행 가능성) 우선으로 부분 수정만 수행했고, 전체 UI 리팩터링은 범위가 커서 별도 단계가 필요합니다.

## 2) 시드 데이터 계정 비밀번호 운영 편차
- 정책성 기대: 기본 계정으로 즉시 로그인 가능한 운영 체크.
- 현황: `pupoo_seed_v5_final.sql`에는 비밀번호 해시만 있으며 평문 비밀번호가 제공되지 않습니다.
- 조치: 로컬 검증은 회원가입 플로우로 신규 계정을 생성해 진행하도록 문서화했습니다.
