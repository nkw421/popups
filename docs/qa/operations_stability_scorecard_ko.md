# 운영 안정도 산정표

## 목적

- `운영 안정도 94%`처럼 넓은 평균 수치가 실제 핵심 사용자 경험을 가리는 문제를 줄인다.
- 인프라 헬스, 배포 성공, 로그 청결도뿐 아니라 `회원가입`, `로그인`, `결제` 같은 핵심 사용자 여정을 별도 가중치로 반영한다.
- 점수만 보지 않고, 왜 그 점수가 나왔는지 근거를 함께 남긴다.

## 기본 원칙

- 총점은 `100점 만점`으로 계산한다.
- 점수는 `현재 시점의 운영 상태`와 `최근 24시간~7일의 운영 이력`을 함께 반영한다.
- 핵심 사용자 여정 장애가 있었다면 인프라 점수가 높아도 총점을 강제로 제한한다.

## 핵심 사용자 여정

- 회원가입 시작: `POST /api/auth/signup/start`
- 회원가입 완료: `POST /api/auth/signup/complete`
- 로그인: `POST /api/auth/login`
- 토큰 갱신: `POST /api/auth/refresh`
- 결제 준비/승인: KakaoPay ready, approve

## 점수표

### 1. 인프라 상태 `25점`

- 백엔드 Deployment Ready, AI Deployment Ready, Redis Ready
- HPA가 상한에 붙어 있지 않고 CPU가 안정 범위 안에 있음
- `api/health`, `ai/health`, 대표 웹 진입점이 정상 응답

권장 배점:

- 백엔드 Ready: `10점`
- AI/Redis Ready: `5점`
- HPA 여유: `5점`
- 헬스 체크 정상: `5점`

### 2. 배포 신뢰도 `15점`

- 최근 배포 워크플로우 성공률
- 최신 기대 SHA와 실제 운영 Deployment 이미지 일치 여부
- 배포 직후 롤백 없이 정상 정착 여부

권장 배점:

- 최근 5회 배포 성공률: `10점`
- 최신 SHA 반영 일치: `5점`

### 3. 런타임 오류 청결도 `20점`

- 최근 12시간 백엔드 로그 기준
- `Unhandled exception`
- `ClassCastException`
- `Internal server error`

권장 배점:

- 치명적 예외 0건: `10점`
- 500성 오류 0건: `10점`

### 4. 핵심 사용자 여정 안정도 `30점`

- 회원가입 시작
- 회원가입 완료
- 로그인/리프레시
- 결제 준비/승인

권장 배점:

- 회원가입 시작: `8점`
- 회원가입 완료: `8점`
- 로그인/리프레시: `7점`
- 결제 준비/승인: `7점`

판정 기준:

- 실제 운영 API 검증 또는 당일 smoke 통과: 만점
- 최근 24시간 내 장애가 있었으나 현재 해결되고 재검증 완료: `70~85%`
- 최근 24시간 내 장애가 있고 아직 재검증 미완료: `0~60%`

### 5. 외부 연동 정합성 `5점`

- Google OAuth
- KakaoPay
- 도메인/redirect URI/callback 정합성

권장 배점:

- 당일 또는 최근 정기 점검 완료: `5점`
- 체크리스트는 있으나 당일 실검증 없음: `3~4점`

### 6. 관측 가능성/운영 준비도 `5점`

- 운영 지표 API 존재
- 로그 조회 가능
- smoke/checklist 문서 존재
- 핵심 여정별 별도 지표 정의 여부

권장 배점:

- 지표/로그/문서 모두 존재: `3점`
- 핵심 여정별 SLO 또는 점검 기준 존재: `2점`

## 캡 규칙

다음 규칙은 총점 계산 뒤 마지막에 적용한다.

- 최근 24시간 내 `핵심 사용자 여정` 장애가 1건 이상 있었으면 총점 상한을 `92점`으로 제한한다.
- 최근 24시간 내 `핵심 사용자 여정` 장애가 아직 미해결이면 총점 상한을 `85점`으로 제한한다.
- 최근 7일 내 같은 핵심 여정 장애가 2회 이상 반복되면 총점 상한을 `88점`으로 제한한다.

이 규칙이 필요한 이유:

- 인프라가 멀쩡해도 회원가입이나 결제가 깨지면 사용자가 느끼는 안정도는 급격히 낮아진다.
- 평균 기반 점수가 핵심 플로우 장애를 가려서는 안 된다.

## 점수 해석

- `95~100`: 매우 안정적. 핵심 사용자 여정까지 최근 검증 완료
- `90~94`: 안정적이지만 최근 핵심 흐름 이슈 또는 미검증 영역이 있음
- `85~89`: 운영 가능하지만 핵심 흐름 리스크가 남아 있음
- `80~84`: 주의 필요. 사용자 체감 장애 가능성이 높음
- `79 이하`: 불안정. 우선순위 장애 대응 필요

## 현재 결론

- `94점`이라고 바로 말하는 것보다, `2026-03-29 기준 운영 안정도는 91점`이라고 보는 편이 더 정직하다.
- 인프라와 배포는 매우 안정적이다.
- 다만 핵심 사용자 여정에서 당일 실제 장애가 있었기 때문에, 체감 안정도까지 포함하면 90점 초반으로 보는 것이 적절하다.

- `운영 안정도`와 `인프라 헬스 점수`를 서로 다른 숫자로 분리해 표현
