============================================================
1) 먼저 결론: 이 프로젝트의 LSTM은 어떤 형태인가?
============================================================
- "딥러닝 LSTM 네트워크를 end-to-end로 직접 학습"한 구조가 아니다.
- 구현은 다음 2단계 하이브리드다.
  1) 고정 게이트 값을 갖는 경량 LSTM-style 계산으로 raw 점수 생성
  2) Ridge(선형회귀 + L2 정규화)로 raw 점수를 보정(calibration)
- 따라서 학습되는 핵심 파라미터는 LSTM 게이트가 아니라, 보정 계수(coefficients/intercept)다.

이 설계의 의도(코드 구조 기반 해석)
- 실서비스에서 빠른 추론, 낮은 복잡도, 높은 안정성을 확보하려는 목적.
- LightGBM을 주력으로 두고 LSTM은 보조 신호/스무딩 역할을 수행.

============================================================
2) 입력 데이터와 전처리
============================================================
- 입력 길이: 60개 시퀀스(SEQUENCE_LENGTH=60)
- 각 값 범위: 0~100 점수로 클램프
- 입력 형식: 숫자 또는 {score|congestion_score|value|v} 키를 가진 객체
- 유효하지 않은 길이/값이면 예측을 중단(None 반환)

실무적 의미
- API 입력이 흔들려도 안전하게 실패(fail-safe)한다.
- 모델이 비정상 입력으로 오염되는 것을 차단한다.

============================================================
3) LSTM 스타일 raw 점수 계산 원리
============================================================
핵심 상태 변수
- h: hidden state
- c: cell state

정규화
- x_norm = (x / 50) - 1  (0~100 -> -1~1 변환)

시점별 게이트 계산(고정 계수)
- forget_gate = sigmoid(0.95*x + 0.20*h + 0.20)
- input_gate  = sigmoid(0.80*x + 0.30*h - 0.10)
- output_gate = sigmoid(0.90*x + 0.15*h - 0.15)
- candidate   = tanh(0.85*x + 0.25*h)

상태 업데이트
- c = forget_gate*c + input_gate*candidate
- h = output_gate*tanh(c)

시퀀스 60개를 순차 처리한 후
- current_score = clamp100((h + 1) * 50)

해석 포인트
- forget/input/output 게이트 덕분에 "최근 변화 반영"과 "과거 맥락 유지"를 동시에 처리한다.
- 다만 계수가 고정이므로, 데이터에 맞춰 LSTM 내부 구조가 자동 최적화되지는 않는다.

============================================================
4) 향후 60분 예측값(raw_avg) 생성 방식
============================================================
수평선 예측 범위
- FORECAST_HORIZON = 12 (5분 간격이므로 60분)

추세 반영
- slope = (sequence[-1] - sequence[-12]) / 11

롤링 예측(12회 반복)
- rolling = clamp100(rolling*0.88 + (rolling + slope*1.4)*0.12)
- 매 스텝 rolling을 forecast 목록에 저장

최종 raw LSTM 평균
- raw_avg = forecast 12개 평균

의미
- 최근 기울기(slope)를 약하게 주입해 단기 추세를 반영한다.
- 0.88/0.12 혼합으로 급격한 변동을 완화해 안정적 평균 예측을 만든다.

============================================================
5) Calibration(보정) 단계: Ridge 회귀
============================================================
왜 필요한가
- raw LSTM 점수는 단순/보수적인 경향이 있어 실제 레이블과 오차가 발생할 수 있다.
- 이를 데이터 기반 선형 보정으로 교정한다.

보정 피처 8개(_FEATURE_COUNT=8)
1. raw_score
2. 최근 12개 평균
3. 최근 12개 표준편차
4. 최근 12개 기울기
5. 최근 30개 평균
6. 최근 12개 범위(max-min)
7. 마지막 값
8. 첫 값

보정 수식
- calibrated = dot(feature, coefficients) + intercept
- 최종값은 0~100 클램프

학습 방식
- train_lstm_calibration_artifact에서 Ridge(alpha=0.8) 학습
- validation_ratio(기본 0.2)로 train/valid 분리
- 산출 artifact:
  - coefficients, intercept
  - metrics(MAE/RMSE, baseline 대비)
  - trainedAt, featureVersion

6) 서비스에서 실제로 어떻게 사용되는가
============================================================
Python AI 서비스(predict_event / predict_program)
1) LSTM_REGISTRY.predict(target_type, inputSequence) 호출
2) lstmPredictedAvgScore를 결과에 포함
3) 이벤트 예측에서는 컨텍스트 보정 단계에서 LSTM avg도 함께 블렌딩

Java 백엔드(AiCongestionService)
1) 응답에 LSTM 타임라인이 없으면, LightGBM 타임라인 형태를 참고해 LSTM 타임라인을 합성
2) 합성 규칙:
   - lightDelta = pointScore - lightAvg
   - lstmScore = clamp(lstmAvg + lightDelta*0.35)
3) 주석 그대로 "LightGBM보다 더 부드럽게(smoother) 유지"가 목적

결론
- LSTM은 최종 단독 의사결정 모델이 아니라
  "평균 혼잡도 보조 지표 + 스무딩된 보조 타임라인" 역할을 수행한다.

7) 장점
============================================================
1. 빠름
- 행렬 연산이 작고 모델이 가벼워 실시간 경로에 유리.

2. 안정성
- 입력 검증/클램프/fallback이 명확해 운영 중 장애 전파를 줄임.

3. 결측 내성
- artifact 없어도 raw 추정은 동작 가능.

4. 설명 가능성
- 보정 피처와 선형 계수로 "왜 값이 바뀌었는지" 설명 가능.

5. 주력 모델 보완
- LightGBM 결과를 보조/검증하는 이중 안전장치로 활용 가능.

8) 한계
============================================================
1. 표현력 한계
- 내부 LSTM 게이트 계수가 고정이라 복잡한 패턴 학습이 제한됨.

2. avg 중심
- 주로 평균 점수 보정에 초점, 피크/급등 상황 표현이 약할 수 있음.

3. 랜덤 분할 검증
- 시계열 누수 가능성 관점에서 time-split 검증이 더 적합할 수 있음.

4. 이벤트 유형 일반화 한계
- 전시/체험/굿즈형 등 도메인별 미세 특성 반영이 제한적.

============================================================
9) 발표/보고서에서 말하면 좋은 핵심 문장
============================================================
- 본 LSTM 모듈은 고정 LSTM-style 시계열 추정과 Ridge 기반 보정으로 구성된 경량 보조 모델이다.
- 주력 LightGBM 예측을 보완하고, 변동성이 큰 실시간 환경에서 안정적 평균 혼잡도 지표를 제공하도록 설계되었다.
- 특히 백엔드에서는 LSTM을 더 부드러운 타임라인 신호로 재구성해 사용자에게 과도하게 출렁이지 않는 예측을 제공한다.
